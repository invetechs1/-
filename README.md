# نظام عزوم للعروض الفنية والمالية 🏗️

**الذراع التنفيذي لشركة عزوم في إعداد العروض الفنية والمالية والخطط التنفيذية** — ارفع ملفات المشروع (كراسة الشروط، RFP، جداول الكميات) ويتولى النظام إعداد عرض احترافي متكامل جاهز للتقديم للعملاء والجهات الحكومية عبر منصة اعتماد.

## ✨ القدرات الرئيسية

| القدرة | التفاصيل |
|---|---|
| **توليد العرض الفني** | وفق الهيكل المعتمد في المنافسات الحكومية السعودية: ملخص تنفيذي، فهم نطاق العمل، المنهجية، فريق العمل، الجودة، السلامة، المخاطر، المحتوى المحلي والسعودة، مصفوفة الالتزام بالمتطلبات |
| **توليد العرض المالي** | جدول كميات مُسعَّر من قاعدة أسعار عزوم + نموذج تسعير كامل (تكلفة مباشرة → إدارية وعمومية → احتياطي مخاطر → هامش ربح → ضريبة القيمة المضافة 15%) مع حساب الضمان الابتدائي |
| **الخطة التنفيذية** | مراحل بمدد ومخرجات قابلة للقياس + مخطط زمني (Gantt) |
| **قاعدة أسعار حية** | بنود مُكوَّدة بتصنيفات، سجل تاريخي لتغير الأسعار، استيراد/تصدير CSV |
| **أرشيف العروض** | كل العروض السابقة محفوظة بحالاتها (مسودة/مُقدَّم/فائز/غير فائز) كمرجع دائم للتسعير |
| **المكتبة الفنية** | نصوص عزوم المعتمدة (منهجيات، خطط جودة وسلامة...) تُعاد صياغتها في كل عرض |
| **تصدير احترافي** | ملف Word بهوية عزوم (RTL كامل) + جدول كميات Excel بصيغة قابلة للنقل لمنصة اعتماد |
| **خزنة وثائق الشركة** | الوثائق النظامية للتأهل (سجل تجاري، زكاة، تأمينات، سعودة، تصنيف، محتوى محلي...) مع تنبيهات تلقائية قبل انتهاء الصلاحية بـ30 يوماً |
| **تحليلات الفوز/الخسارة** | نسبة الفوز وقيمه حسب الجهة والعميل + مقارنة هوامش الربح بين العروض الفائزة والخاسرة لمعايرة التسعير |
| **المستودع المعرفي** | ارفع أي عروض قديمة (لعزوم أو لشركات منافسة) — يكتشف النظام أعمدة جدول الكميات تلقائياً ويستخرج البنود كأسعار سوق للمقارنة، ويغذي بمحتواها توليد العروض القادمة |
| **محلل فرصة الفوز** | قرار تقدّم/لا تتقدّم مسبب قبل إعداد العرض: خبرة مماثلة، تغطية تسعير، جاهزية وثائق، سجل العميل، ورصد اشتراطات التأهيل عالية الخطورة في كراسة الشروط |
| **الذاكرة المرجعية** | كل عرض سابق يُحلل ويُحفظ، والمشروع الجديد يُبنى تلقائياً من أقرب العروض السابقة لنطاقه |
| **ذكاء اصطناعي** | تحليل وثائق المشروع وتوليد العرض عبر Claude API — مع محرك قوالب ذكي يعمل بدون إنترنت |

## 🚀 التشغيل

```bash
./run.sh
```

ثم افتح المتصفح على: **http://localhost:8000**

### تفعيل التوليد بالذكاء الاصطناعي (اختياري لكنه موصى به)

```bash
cp .env.example .env
# ضع مفتاح Claude API في المتغير ANTHROPIC_API_KEY داخل ملف .env
```

بدون المفتاح يعمل النظام بمحرك القوالب الذكية (مطابقة كلمات مفتاحية مع قاعدة الأسعار + نصوص المكتبة الفنية) — العرض الناتج كامل وقابل للتحرير، لكن التوليد بـ Claude يقرأ الوثائق فعلياً ويستخلص النطاق والكميات ومصفوفة الالتزام بدقة أعلى بكثير.

## 📖 سير العمل

1. **حدّث قاعدة الأسعار** — البيانات الأولية استرشادية؛ استبدلها بأسعار عزوم الفعلية (يدوياً أو باستيراد CSV).
2. **حدّث المكتبة الفنية** — خصوصاً «نبذة عن شركة عزوم» و«الخبرات والمشاريع المماثلة».
3. **اضبط الإعدادات** — بيانات الشركة والنسب المالية (الإدارية %، المخاطر %، الربح %).
4. **أنشئ عرضاً جديداً** — ارفع ملفات المشروع واضغط توليد.
5. **راجع وحرّر** — عدّل الكميات والأسعار في جدول الكميات (يُعاد الحساب فورياً).
6. **صدّر** — Word للعرض الكامل + Excel لجدول الكميات، وقدّمهما عبر اعتماد أو مباشرة للعميل.
7. **حدّث حالة العرض** — عند الترسية حدّثها إلى «فائز» ليبقى الأرشيف مرجعاً لأسعارك الرابحة.

## 🏛️ ملاءمة المنافسات الحكومية (منصة اعتماد)

- هيكل العرض الفني يطابق ما تقيّمه لجان الفحص وفق نظام المنافسات والمشتريات الحكومية ولوائحه.
- ضريبة القيمة المضافة تُعرض في بند مستقل وفق الممارسة السعودية.
- الضمان الابتدائي يُحسب تلقائياً (الافتراضي 1% — عدّله من الإعدادات حسب شروط كل منافسة).
- سريان العرض الافتراضي 90 يوماً.
- جدول الكميات يُصدَّر Excel لإعادة إدخاله في نموذج اعتماد.

```
├── export_xlsx.py       # تصدير جدول الكميات Excel
└── static/              # الواجهة العربية
```

## 🏛️ الربط مع منصة اعتماد

صفحة «منافسات اعتماد» تنفذ الدورة المطلوبة:

1. **جلب المنافسات** — زر واحد يسحب المنافسات المطروحة من واجهة الزوار العامة في `tenders.etimad.sa` (لا يحتاج تسجيل دخول) ويرتبها بدرجة **ملاءمة لنشاط عزوم** محسوبة من ذاكرتك المرجعية، مع رابط مباشر لصفحة كل منافسة.
2. **الموافقة** — غيّر حالة المنافسة إلى «مهتمون»، واضغط «أنشئ عرضاً» فتنتقل بياناتها لشاشة التوليد.
3. **كراسة الشروط** — نزّلها من اعتماد وارفعها في شاشة التوليد: يحلل النظام الفرصة، يقرأ الاشتراطات، ويبني العرض الفني والمالي بهوية عزوم.
4. **الرفع** — تصدير Word للعرض الفني + Excel لجدول الكميات جاهزين للرفع في المنصة.

### دخول نفاذ (لتنزيل الكراسات آلياً لاحقاً)

```bash
pip install playwright && playwright install chromium
python scripts/etimad_nafath_login.py
```

يفتح متصفحاً على دخول اعتماد، يُدخل رقم هويتك (من شاشة الإعدادات — **يُحفظ محلياً في قاعدة بياناتك فقط ولا يُرفع لأي مكان**)، يظهر رقم نفاذ توافق عليه من جوالك، وتُحفظ الجلسة في `data/etimad_cookies.json`.

> ⚠️ شغّل الجلب من جهازك — بعض البيئات السحابية تحجب نطاق اعتماد. وأسماء حقول المنصة قد تتغير مع تحديثاتها؛ الوحدة مكتوبة بقراءة مرنة للحقول وتعرض رسالة واضحة عند أي تغيير.

### تطويرات أخرى مقترحة
- تنزيل كراسات الشروط آلياً بعد الدخول وربطها بشاشة التوليد مباشرة.
- مسار اعتماد داخلي (مراجعة فنية ← اعتماد الإدارة للسعر) قبل التقديم.
- تفصيل أسعار البنود إلى مواد/عمالة/معدات/مقاولي باطن مع مؤشرات تصعيد الأسعار.
